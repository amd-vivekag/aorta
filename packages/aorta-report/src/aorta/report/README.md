## 2. Installation

### Prerequisites

- Python >= 3.10

### Install with pip

**Option 1: Install from source (recommended)**

```bash
# Navigate to the aorta package directory
cd /path/to/aorta

# Install the package with report dependencies
pip install -e packages/aorta-report/
# Or install from GitHub
pip install "git+https://github.com/ROCm/aorta.git#subdirectory=packages/aorta-report"
```

**Option 2: Install individual dependencies**

```bash
# Core dependencies
pip install pyyaml>=6.0
pip install click>=8.0.0

# Report dependencies
pip install numpy>=1.20.0
pip install pandas>=1.3.0
pip install openpyxl>=3.0.0
pip install matplotlib>=3.7.0
pip install seaborn>=0.12.0
pip install beautifulsoup4>=4.12.0

# TraceLens (for GEMM analysis)
pip install git+https://github.com/AMD-AGI/TraceLens.git
```

### Install with uv (faster alternative)

```bash
cd /path/to/aorta

# Install the package with report dependencies
uv pip install -e packages/aorta-report/

# Or install all packages from workspace
uv sync --all-packages
```

### Verify Installation

```bash
aorta-report --version
aorta-report --help
```

## 3. Pipeline Commands

### GEMM Variance Analysis Pipeline

Analyzes GEMM kernel time variance across thread/channel configurations.

```bash
aorta-report pipeline gemm \
    --sweep-dir ./experiments/2026-01-10/gemm-sweep/ \
    -o ./comparison_gemm/
```

**Options:**

| Option | Description | Default |
|--------|-------------|---------|
| `--sweep-dir` | Path to sweep directory with thread/channel subdirectories | Required |
| `-o, --output` | Output directory for results | Required |
| `--skip-tracelens` | Skip TraceLens analysis if reports already exist | `False` |
| `--top-k` | Number of top GEMM kernels to extract | `5` |
| `-t, --threads` | Thread configs to analyze (can specify multiple) | `256, 512` |
| `-c, --channels` | Channel configs to analyze (can specify multiple) | `28, 42, 56, 70` |
| `--no-plots` | Skip plot generation | `False` |
| `--no-html` | Skip HTML report generation | `False` |

**Example with options:**

```bash
aorta-report pipeline gemm \
    --sweep-dir ./experiments/2026-01-10/gemm-sweep/ \
    -o ./comparison_gemm/ \
    --skip-tracelens \
    --top-k 10 \
    -t 256 -t 512 \
    -c 28 -c 56
```

---

### Summary Comparison Pipeline

Compares two configurations (baseline vs test) with comprehensive analysis.

```bash
aorta-report pipeline summary \
    --baseline ./experiments/2026-01-10/rccl-warp-speed/32cu_512threads/ \
    --test ./experiments/2026-01-10/rccl-warp-speed/37cu_384threads/ \
    --baseline-label 32c_512t \
    --test-label 37c_384t \
    --output ./comparison_rccl/
```

**Options:**

| Option | Description | Default |
|--------|-------------|---------|
| `--baseline` | Path to baseline trace directory | Required |
| `--test` | Path to test trace directory | Required |
| `--baseline-label` | Label for baseline in reports | Required |
| `--test-label` | Label for test in reports | Required |
| `--output` | Output directory for results | Required |
| `--skip-tracelens` | Skip TraceLens analysis if reports already exist | `False` |
| `--gpu-timeline/--no-gpu-timeline` | Include GPU timeline comparison | `True` |
| `--collective/--no-collective` | Include collective/NCCL comparison | `True` |

**Example with options:**

```bash
aorta-report pipeline summary \
    --baseline ./experiments/2026-01-10/rccl-warp-speed/32cu_512threads/ \
    --test ./experiments/2026-01-10/rccl-warp-speed/56cu_256threads/ \
    --baseline-label baseline_32cu \
    --test-label test_56cu \
    --output ./comparison_output/ \
    --skip-tracelens
```

## 5. Tips

1. **First run**: Let the pipeline run TraceLens (don't use `--skip-tracelens`)
2. **Subsequent runs**: Use `--skip-tracelens` to save time if TraceLens reports already exist
3. **Large datasets**: Use `--no-plots --no-html` for faster processing when only data files are needed
4. **Custom analysis**: Use the CSV/Excel outputs for custom visualization in your preferred tools
5. **Sharing results**: HTML reports are self-contained and can be shared directly with your team

    --test-label "Test" \
